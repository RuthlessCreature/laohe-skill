# Issue Severity Rules

## Blocker

满足任一条件：

- 核心功能不可用。
- 数据丢失、数据错写、状态不可恢复。
- 权限绕过 / 越权操作。
- 无法启动 / 无法部署 / 无法登录。
- 安全风险：敏感信息泄露、未授权访问。
- 影响主流程且没有可接受 workaround。

处理规则：

- 必须修复或明确取消本轮发布。
- Open Blocker 禁止进入 RC。

## Major

满足任一条件：

- 重要功能异常，但存在临时 workaround。
- 状态流转错误但可恢复。
- 关键页面交互明显错误。
- API 行为和 SPEC 不一致。
- 自动化测试覆盖核心链路失败。

处理规则：

- 默认必须修复。
- 如不修复，必须有业务接受风险。

## Minor

满足任一条件：

- 文案、样式、非核心交互问题。
- 不影响主流程的小兼容问题。
- 日志、提示、排序、展示细节问题。

处理规则：

- 可带风险进入 RC，但必须记录。

## Suggestion

- 可优化项。
- 非本轮需求。
- 技术债建议。

处理规则：

- 不阻塞发布。
